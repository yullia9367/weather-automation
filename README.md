# weather-automation — 알렌(Allen, 75013) 날씨 자동 기록 & 시각화 만들기

> 미국 텍사스 알렌(Allen, TX, 우편번호 75013) 지역의 날씨를 매일 오전 6시에 **자동으로** 가져와
> 텍스트 파일에 기록하고, 보기 좋은 **HTML 카드**로 시각화하는 프로그램을
> 처음부터 끝까지 만든 과정을 정리한 문서입니다.
> 프로그래밍을 처음 접하는 분도 따라올 수 있도록 최대한 쉽게 설명했습니다.

## 1. 이 프로그램이 하는 일 (한눈에 보기)

```
   ┌─────────────┐   매일 오전 6시   ┌──────────────┐   기록   ┌──────────────────┐
   │  macOS의     │  ───────────────▶ │  weather.py   │ ───────▶ │ weather_log.txt   │
   │  자동 실행기  │   자동으로 실행     │ (파이썬 코드)  │          │ (날씨 기록 파일)   │
   │  (launchd)   │                   └──────────────┘          └──────────────────┘
   └─────────────┘                          │
                                            │ 인터넷에서 날씨 받아오기
                                            ▼
                                    ┌─────────────────┐
                                    │  Open-Meteo API   │  ← 무료 날씨 서비스
                                    └─────────────────┘

   그리고 기록된 데이터를 예쁘게 보여주는 화면:

                       weather.html  ← 웹 브라우저로 열어서 보는 카드형 날씨
```

핵심은 3가지입니다.

1. **날씨 가져오기** — 인터넷의 무료 날씨 서비스에서 알렌의 일주일 예보를 받아옵니다.
2. **자동 실행** — 사람이 손대지 않아도 매일 아침 6시에 알아서 실행됩니다.
3. **시각화** — 받아온 데이터를 웹 페이지(HTML)로 예쁘게 보여줍니다.

## 2. 폴더 구조

프로그램은 `~/weather-alarm/` 폴더 안에 들어 있습니다.

```
weather-alarm/
├── weather.py            ← 날씨를 가져와 기록하는 파이썬 프로그램 (핵심)

├── weather.html          ← 날씨를 예쁘게 보여주는 웹 페이지
```

그리고 자동 실행을 담당하는 설정 파일이 macOS의 정해진 위치에 하나 더 있습니다.

```
~/Library/LaunchAgents/com.yullia9367.weather-alarm.plist   ← 매일 6시 실행 예약표
```

## 3. 만드는 과정 (단계별)

### 단계 1 — "어디의 날씨를?" 위치 정하기

날씨를 받으려면 **위도(latitude)** 와 **경도(longitude)** 라는 좌표가 필요합니다.
지구 위 한 점을 숫자 두 개로 나타내는 주소라고 생각하면 됩니다.

- 지역: 알렌 (Allen, 텍사스, 우편번호 75013)
- 위도: `33.10`
- 경도: `-96.67`  (미국은 서반구라 경도가 음수입니다)

파이썬 코드에서는 이렇게 맨 위에 상수로 적어두었습니다.

```python
LATITUDE = 33.10
LONGITUDE = -96.67
LOCATION_NAME = "알렌"
```

> 💡 다른 도시로 바꾸고 싶다면 이 세 줄의 숫자만 바꾸면 됩니다.

---

### 단계 2 — 날씨 데이터 받아오기 (Open-Meteo API)

날씨 정보는 **Open-Meteo**라는 무료 서비스에서 가져옵니다.
회원가입도, API 키도 필요 없어서 초보자에게 딱 좋습니다.

우리가 요청하는 것:
- 일별 **최고 기온**, **최저 기온**, **날씨 상태 코드**
- 앞으로 **7일치** 예보

```python
def fetch_weather():
    params = {
        "latitude": LATITUDE,
        "longitude": LONGITUDE,
        "daily": "temperature_2m_max,temperature_2m_min,weather_code",
        "timezone": "auto",
        "forecast_days": 7,
    }
    url = "https://api.open-meteo.com/v1/forecast?" + urllib.parse.urlencode(params)
    with urllib.request.urlopen(url, timeout=15) as resp:
        return json.loads(resp.read().decode("utf-8"))
```

**쉽게 풀어보면:**
- `params` = "이런이런 조건으로 날씨 알려줘"라고 적은 요청서
- `url` = 그 요청서를 인터넷 주소로 만든 것
- `urlopen(...)` = 인터넷에 실제로 접속해서 답을 받아오는 부분
- `json.loads(...)` = 받아온 답(글자 덩어리)을 파이썬이 다루기 쉬운 형태로 변환

> 이 프로그램은 파이썬에 기본으로 들어있는 도구(`urllib`)만 사용합니다.
> 별도 라이브러리 설치가 필요 없습니다.

### 단계 3 — 날씨 코드를 사람 말로 바꾸기

API는 날씨를 **숫자 코드**로 알려줍니다. 예를 들어 `0`은 맑음, `3`은 흐림입니다.
(이것을 WMO 날씨 코드라고 부릅니다.)

사람이 읽기 쉽도록 코드 → 한글 사전을 만들어 두었습니다.

```python
WEATHER_CODES = {
    0: "맑음",
    1: "대체로 맑음",
    2: "부분적으로 흐림",
    3: "흐림",
    61: "약한 비",
    80: "약한 소나기",
    95: "천둥번개",
    # ... (실제 코드에는 더 많이 들어있음)
}
```

이렇게 하면 `2`라는 숫자를 받았을 때 `"부분적으로 흐림"`으로 바꿔 보여줄 수 있습니다.

### 단계 4 — 섭씨를 화씨로도 함께 표시하기

알렌은 미국 도시라서 화씨(°F)도 익숙합니다. 그래서 섭씨를 화씨로 바꾸는
간단한 계산 함수를 넣었습니다.

```python
def c_to_f(celsius):
    return celsius * 9 / 5 + 32
```

예: 38°C → 100.4°F

### 단계 5 — 보기 좋게 정리해서 출력하고 파일에 기록하기

받아온 7일치 데이터를 한 줄씩 표처럼 정리합니다.

```python
for date, t_max, t_min, code in zip(dates, maxes, mins, codes):
    desc = WEATHER_CODES.get(code, f"알 수 없음(코드 {code})")
    ...
    lines.append(
        f"  {date_str}{marker}  |  {desc:<10}  |  "
        f"최고 {t_max:>5.1f}°C / {c_to_f(t_max):>5.1f}°F  |  "
        f"최저 {t_min:>5.1f}°C / {c_to_f(t_min):>5.1f}°F"
    )
```

- `zip(...)` = 날짜, 최고기온, 최저기온, 날씨코드를 하루 단위로 짝지어 묶어줍니다.
- 오늘 날짜에는 `[today]` 표시를 붙입니다.

그리고 결과를 **화면에 출력**하는 동시에 **`weather_log.txt` 파일 끝에 계속 이어서 기록**합니다.

```python
with open(LOG_FILE, "a", encoding="utf-8") as f:
    f.write(output)
    f.write("\n\n")
```

> 여기서 `"a"`는 **append(이어쓰기)** 모드입니다.
> 매일 실행할 때마다 기존 내용을 지우지 않고 아래에 계속 쌓입니다.
> 그래서 시간이 지나면 날씨 변화의 "기록장"이 됩니다.

실제로 기록된 모습(`weather_log.txt`):


### 단계 6 — 매일 자동으로 실행되게 만들기 (launchd)

사람이 매일 아침 프로그램을 직접 실행하는 건 번거롭습니다.
그래서 macOS에 기본으로 들어있는 **launchd**라는 자동 실행 도구를 이용합니다.
(윈도우의 "작업 스케줄러", 리눅스의 "cron"과 같은 역할)

`~/Library/LaunchAgents/com.yullia9367.weather-alarm.plist` 파일에
"매일 오전 6시 0분에 weather.py를 실행하라"는 예약표를 적어둡니다.

```xml
<key>ProgramArguments</key>
<array>
    <string>/usr/bin/python3</string>
    <string>/Users/yullia9367/weather-alarm/weather.py</string>
</array>

<key>StartCalendarInterval</key>
<dict>
    <key>Hour</key>
    <integer>6</integer>
    <key>Minute</key>
    <integer>0</integer>
</dict>
```

**쉽게 풀어보면:**
- `ProgramArguments` = "파이썬으로 weather.py를 실행해라"
- `StartCalendarInterval` = "매일 6시 0분에"
- 실행 결과는 `launchd_stdout.log`, 에러는 `launchd_stderr.log`에 저장됩니다.

#### 자동 실행 등록/해제 방법

```bash
# 등록(활성화)
launchctl load ~/Library/LaunchAgents/com.yullia9367.weather-alarm.plist

# 해제(비활성화)
launchctl unload ~/Library/LaunchAgents/com.yullia9367.weather-alarm.plist

# 지금 당장 한 번 실행해서 테스트
launchctl start com.yullia9367.weather-alarm
```

> ⚠️ 참고: launchd는 컴퓨터가 **켜져 있고 로그인된 상태**여야 예약 시각에 실행됩니다.
> 예약 시각에 컴퓨터가 꺼져 있었다면, 다음에 켜졌을 때 밀린 작업을 실행합니다.

### 단계 7 — 날씨를 예쁘게 시각화하기 (weather.html)

텍스트 기록만으로는 밋밋하니, 웹 브라우저로 열면 예쁜 **카드 형태**로
보여주는 `weather.html`을 만들었습니다.

특징:
- 하루하루가 **카드 한 장**으로 표시됩니다.
- 날씨에 맞는 **이모지 아이콘**(⛅ ☁️ 🌦️ 🌧️)이 들어갑니다.
- **오늘 날짜** 카드는 파란색으로 강조되고 `TODAY` 배지가 붙습니다.
- 최고 기온은 빨간색, 최저 기온은 파란색으로 구분됩니다.
- 화면 크기에 따라 카드가 자동으로 줄바꿈되는 **반응형** 디자인입니다.

여는 방법:
```bash
open ~/weather-alarm/weather.html
```
또는 파인더에서 `weather.html`을 더블클릭하면 브라우저로 열립니다.

> 💡 현재 `weather.html`은 특정 날짜의 데이터가 직접 적혀 있는 형태입니다.
> 향후 개선한다면, `weather.py`가 실행될 때마다 최신 데이터를 자동으로
> HTML에 채워 넣도록 만들 수 있습니다. (아래 "더 해보면 좋은 것" 참고)

## 4. 직접 실행해보기

자동 실행을 기다리지 않고 수동으로 테스트하고 싶다면:

```bash
# 폴더로 이동
cd ~/weather-alarm

# 프로그램 실행
python3 weather.py
```

실행하면 화면에 일주일 날씨가 표로 출력되고,
동시에 `weather_log.txt` 파일에도 같은 내용이 추가됩니다.

## 5. 자주 겪는 문제 (초보자 체크리스트)

| 증상 | 원인 / 해결 |
|------|-------------|
| "날씨 정보를 가져오지 못했습니다" | 인터넷 연결 확인. 잠시 후 다시 실행. |
| 자동 실행이 안 됨 | `launchctl load` 로 등록했는지 확인. 예약 시각에 컴퓨터가 켜져 있었는지 확인. |
| 한글이 깨져 보임 | 파일을 UTF-8 인코딩으로 여는지 확인 (코드에는 이미 반영됨). |
| 로그가 너무 커짐 | `weather_log.txt`는 계속 쌓이므로, 가끔 내용을 비우거나 백업하세요. |
| 에러 원인을 모르겠음 | `launchd_stderr.log` 파일을 열어보면 에러 메시지가 있습니다. |

## 6. 이 프로젝트에서 배우는 것

- **API**: 인터넷 서비스에서 데이터를 받아오는 방법 (Open-Meteo)
- **파일 입출력**: 데이터를 파일에 기록하고 쌓아두는 방법 (`open`, `"a"` 모드)
- **자동화**: launchd로 프로그램을 정해진 시각에 자동 실행하는 방법
- **데이터 변환**: 숫자 코드 → 한글 설명, 섭씨 → 화씨
- **시각화**: 데이터를 HTML/CSS로 보기 좋게 표현하는 방법
